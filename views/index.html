The Exercise Tracker app can be used to track the exercise history -including type, duration and date- of a given user. Use the above submitting forms to create a new user or to add exercise details to an existent one.

  • I can POST a form data username to [project_url]/api/exercise/new-user. In return I will receive a JSON response containing the username and _id.
    Example: {username:"Theusername", _id:1424ds}
  • I can use the API end point GET [project_url]/api/exercise/users to obtain an array of all registered users and their _id.
  • I can POST a form data to [project_url]/api/exercise/add to add information to any registered user. The userId(_id), exercise and duration are mandatory fields while date is optional. If date is not supplied, the app will use the date at the moment of submission.
  • I can use the API end point GET [project_url]/api/exercise/log with a parameter of userId(_id) to retrieve the user object with added array log and count (total exercise count).
  • The POST of the form exercise will return a JSON response containing all the information for the given userId(_id).
  • I can retrieve part of the log of any user by also passing along optional parameters of from & to or limit. (Date format yyyy-mm-dd, limit = int).
    GET [project_url]/api/exercise/log?{userId}[&from][&to][&limit]
    { } = required, [ ] = optional
    from, to = dates (yyyy-mm-dd); limit = integer number
